# Nexa Positioning And Messaging

## One-Line Positioning

`Nexa is a voice-first AI workspace and browser assistant that helps builders research, plan, and execute faster.`

## Short Product Description

`Nexa combines an AI workspace with a browser-native assistant so users can think, research, compare, organize, and act in one place.`

## Core Product Structure

- `Nexa AI`: the workspace assistant for chat, voice, project memory, files, planning, and research
- `Nexa Browser`: the browser environment where Nexa can see pages, summarize, compare, extract, and help users act
- `Project Brain`: the project-specific memory layer connecting both products

## The Big Idea

Nexa should not be positioned as another chatbot.

Nexa should be positioned as:

- an AI workspace for builders
- a browser-aware research and execution assistant
- a system that understands projects, not just prompts

The key difference is that Nexa should help users do work inside context, not only generate answers in isolation.

## Brand Thesis

Most AI products are still prompt boxes.

Nexa should feel like:

- a workspace that remembers what the user is building
- a browser assistant that understands what the user is looking at
- a voice-first partner that helps decide what to do next

## Category Definition

Nexa belongs in the category:

- `AI workspace`
- `browser assistant`
- `project-aware productivity AI`

Not:

- generic chatbot
- browser with chat bolted on
- autonomous agent for everything

## Who Nexa Is For

Primary users:

- founders
- developers
- students
- creators
- researchers

Shared need:

They are all trying to turn ideas, documents, websites, and research into output faster.

## Positioning Statement

`For builders who need more than a chatbot, Nexa is a voice-first AI workspace and browser assistant that helps them research the web, organize projects, and get work done faster. Unlike standard chat apps, Nexa understands project context and lives directly inside the browsing workflow.`

## Messaging Pillars

### 1. Voice-First

Users should be able to speak to Nexa naturally, not just type prompts.

Example framing:

- talk to Nexa
- ask Nexa what matters
- use voice to research, plan, and decide faster

### 2. Project-Aware

Nexa should remember work by project, not as one long disconnected chat history.

Example framing:

- every project gets its own memory
- Nexa remembers what you are building
- switch projects without losing context

### 3. Browser-Native

Nexa Browser should make the AI useful where work actually happens: on the web.

Example framing:

- ask about the current page
- compare websites
- save research into a project
- explain docs where you read them

### 4. Action-Oriented

Nexa should help users move from information to output.

Example framing:

- summarize
- extract
- compare
- organize
- draft
- save to project

## What Nexa Should Feel Like

Nexa should feel:

- alive
- useful
- focused
- technical but accessible
- aware of context

It should not feel:

- like a generic assistant skin
- like a copy of ChatGPT
- like a browser with random AI buttons

## Differentiation

### What users already know

Users already know:

- ChatGPT
- Gemini
- Claude
- Copilot

### What Nexa must add

Nexa must combine:

- chat
- voice
- project memory
- browser context
- research workflow
- execution support

That combination is the moat for the first version.

## Product Narrative

The product story should be:

1. Users already use AI to ask questions.
2. But real work happens across tabs, docs, projects, notes, and decisions.
3. Nexa keeps the assistant inside that workflow.
4. Nexa remembers what the user is building.
5. Nexa helps users research, compare, organize, and act faster.

## What Not To Say

Avoid:

- `the best AI in the world`
- `fully autonomous`
- `replaces teams`
- `does everything`

Prefer:

- clear
- specific
- defensible
- workflow-oriented claims

## Recommended Positioning Summary

The final positioning should be:

`Nexa is a voice-first AI workspace and browser assistant for builders, founders, students, and developers. It helps users research the web, organize projects, and get work done faster by combining chat, project memory, and browser-native context.`
